# Bullet Bank

Use this file to assemble targeted resumes. Bullets should be truthful, ATS-friendly, and selected based on the job description.

## Bullet Formula

Action verb + what was built/changed + technical keywords + scope/domain + impact.

Good pattern:

- Built X using Y to support Z.
- Automated X with Y, processing Z scale.
- Secured X by implementing Y across Z workflows.
- Modernized X from legacy Y to standards-compliant Z.

Avoid:

- Vague claims without technology.
- Unsupported metrics.
- Keyword stuffing that does not read naturally.
- Bullets that only list tasks without engineering impact.
